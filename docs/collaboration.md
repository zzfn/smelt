# 协作联机（设计讨论，未动工）

状态：**只有设计，没有代码**。这份文档是想清楚「smelt 的联机该是什么」，落地方案还没排期。

---

## 核心判断

**别做 Live Share 的复制品。**

VS Code Live Share / Zed 那套协作是「多人同编一份代码」，那是编辑器的战场。smelt 的定位里明确写着「人看导航、下指令，不是自己去改代码」——去抢编辑器的活既打不过，也不是我们要解决的问题。

smelt 手里别人没有的牌是 **smeltd：终端会话能脱离人存活**。普通终端一关 GUI 会话就死了，压根不存在「一个能被多人围观、能被交接、能被远程遥控的对象」。

所以 smelt 的联机应该是：**围绕一个活着的 agent 会话组织多个人**，而不是围绕一份代码组织多个人。

## 冷启动约束（团队方向的硬约束）

已确定首要服务对象是**团队同事**。团队功能最大的敌人不是技术，是冷启动：**任何需要「先说服第二个人装上 smelt」才产生价值的功能，大概率死在第一步。**

由此推出 MVP 的硬性条件：**接收方零安装**（浏览器打开即可）。这一条直接筛掉了「作战地图」「会话移交」当 MVP 的可能——它们都要求全员先装。

---

## 架构洞察：四个功能是同一个底座的四张脸

一个会话对外只有三个通道：

| | 画面通道（字节流） | 状态通道（结构化） | 指令通道（写回） |
|---|---|---|---|
| 观战席 | ✅ | ✅ | 只读 ❌ / 接管 ✅ |
| 远程遥控 | 可选 | ✅ | ✅ |
| 作战地图 | ❌ | ✅ ×N | ❌ |
| 联机 review | ❌（要的是 diff） | ✅ | ✅ |
| 任务分发 / 收件箱 | ❌ | ✅ ×N | ✅ |

关键：**大多数功能根本不需要画面通道，但全都需要状态通道。**

而画面通道已经有了（smeltd 的 `snapshot_ansi` + 字节流重放），状态通道恰恰是缺的那个。

> **结论：联机的第一块砖不是网络，是让 smeltd 知道自己在跑什么。**
>
> roadmap 里那条「smeltd 加 JSON-RPC 结构化控制通道：会话状态从『解析字节流猜』
> 升级为『协议事实』」不再是可选的借鉴项，它是这四个功能共同的地基。

---

## 地基：状态通道

### 为什么它在联机场景下是生死线

本机场景里「靠 OSC 标题猜状态」猜错了无所谓——眼睛一扫终端就知道真相。

联机场景下不行。手机上弹一张卡说「agent 在等你批准」，你点了 yes，结果它其实在跑测试——那个 `y` 就喂进了别的地方。**远程的人没有眼睛去纠错，状态必须是事实，不能是猜测。**

### 三个信源，按可信度叠加

1. **OSC 0/2 + 9/777 + 响铃** —— 通用，任何遵守约定的 agent 都有，但只能猜（现状）
2. **Claude Code hook → `smelt-notify` 直写 smeltd socket** —— roadmap 已有此条；这是
   **协议事实**，agent 亲口说的，可信度最高
3. **`~/.claude/projects/*.jsonl` 的 watcher** —— AgentHub 借鉴项，第三方交叉验证

先把 schema 定死，三个信源都往里灌，高可信度覆盖低可信度。

### schema 草案

```rust
session_id, title, cwd, agent_kind,      // claude / codex / ...
state,                                    // Thinking | ExecutingTool | AwaitingApproval
                                          //   | WaitingForUser | Idle | Dead
state_since,                              // 空转多久了 —— 作战地图要
pending_question,                         // 在问什么 —— 远程遥控的命脉
dirty_files: Vec<PathBuf>,                // 撞车预警靠这个
branch, tokens_used,
```

这一份 schema 同时供养下面四个功能。做完它，每个功能都只剩「选订阅哪几个字段 + 画个前端」。

**它单机就能验证**：做完之后自己的会话总览立刻变准，哪怕联机一行没写也不亏。

## 四张脸

### 一、观战席（MVP 首选）

一条链接甩给同事，他**浏览器打开就能看你的 agent 在干活**，不用装任何东西。

- 满足「接收方零安装」，是冷启动的最优解
- 是其他三条的地基：能观战，才谈得上接管、移交、作战地图
- 顺带白送手机遥控的前端

**差异化：观众看到的不该是裸终端。** tmate 能给你字节流，但给不了驾驶舱——agent 现在
是 thinking 还是等审批、token 烧了多少、这一趟改了哪些文件。这些 smelt 都已经有了。
观战席是「**带仪表盘的直播**」，不是「别人的屏幕」。

权限两档：**只读观战** / **举手接管**（同事请求写权限，主人点头后可直接敲键盘进 PTY）。

#### 观战席的其他应用（直播和录像是同一件事，只差一个实时/回放开关）

字节流本来就在磁盘上，所以这些几乎白送：

- **给非工程师看进度**（最被低估的一条）：PM 想知道需求做到哪了，现在得问你、你截图、
  你解释。给他一条常驻只读链接自己看——**把「进度同步」这个纯负担自动化掉**。而且他看
  到的是仪表盘版本（在跑 / 等审批 / 完成），不是看不懂的终端字符。
- **事故复盘与责任追溯**：agent 删了不该删的、推了坏代码——回放能还原它当时想干嘛、谁
  批准的、批的时候屏幕上写着什么。团队里一旦有人被 agent 坑过一次，这条从「有意思」变
  成「必须有」。
- **教学与布道**：录一段「我怎么指挥 agent 重构这个模块」甩出去。AI coding 最稀缺的是
  真实开车示范，不是 prompt 技巧文章。
- **喂给另一个 agent**：回放/状态流本身就是上下文。roadmap 的 **Remix**（换 provider
  继续跑、transcript 当上下文）就是这个思路的雏形——「让 agent B 看看 agent A 为什么
  卡住」是可能的。

### 二、求助广播（第二步，团队场景真正的杀手锏）

AI coding 时代团队里最蠢的浪费：**agent 卡在「等你批准」，而人去开会了。** 一个会话空转
40 分钟，而团队里任何一个有权限的人本来花 3 秒就能点一下。

agent 进入 `AwaitingApproval` → 自动生成分享链接 → 甩进团队群 → 谁有空谁点进去回一句。
**审批阻塞被消灭了。** 建立在观战席之上，几乎不需要新架构，且极好演示。

### 三、远程遥控

**千万别做成「手机上的小终端」**——手机上敲终端是灾难，没人会用第二次。

正确形态是**卡片 + 按钮**：

```
📱 claude 想在 smelt 里执行 `rm -rf target/`
   [ 批准 ]  [ 拒绝 ]  [ 看完整上下文 ]
```

这是「通知 + 一键回复」，不是「远程终端」。而这要求知道**它到底在问什么**——即 schema
里的 `pending_question`。纯字节流只能甩给用户一块终端画面让他自己找光标敲 y，前功尽弃。

Mac 合上盖子 agent 还在跑（smeltd 白给的天赋），这条的价值才成立。

### 四、联机 review（接 roadmap 的「交互式 diff」）

同事在 diff 上划线写评论 → 打包成 prompt 喂回我的 agent。review 的人和开车的人可以不是
同一个：人负责判断，agent 负责改。

**坑（必须先想清楚）：agent 正在跑的时候不能往 PTY 里塞字节。** 那段文本会被当成它当前
那个问题的答案，直接把会话搞乱。

评论必须**排队**，等状态变成 `WaitingForUser` 才注入。UI 要有「已排队 3 条评论，等 agent
空下来发送」的明确反馈，不能假装发出去了。

### 五、作战地图（需全员安装，排后面）

全队 agent 摊在一张看板上：谁在跑什么、谁在等审批、谁跑挂了、谁空转半小时了。

**撞车预警要做就做预测性的。** 事后检测（两个会话 `dirty_files` 有交集 → 报警）很容易但
价值一般——冲突已经发生了，你只是早点知道。

真正值钱的是**在 agent 动手之前拦住它**：Claude Code 的 **PreToolUse hook** 能在工具调用
前介入——agent 说「我要 Edit `foo.rs`」的瞬间，hook 问 smeltd「有别人在动这个文件吗」，
有就拦下来，让它换个活干或等一等。

**冲突从「事后 merge 血案」变成「事前不发生」。** 这条别人做不了：同时需要多会话视图 +
hook 接入 + 文件级 diff，smelt 三样都有（或差一点点）。

### 六、会话移交（oncall 交接班）

「这个 agent 现在归你了」——连着活着的 PTY、完整 transcript、cwd、worktree 一起交出去。
同事 attach 上来时 agent 还活着，上下文一点没丢，不用重新解释任务背景。

**别人抄不了**：普通终端的会话一关 GUI 就死，根本不存在「可移交的对象」这个概念。

### 七、任务分发 / 团队收件箱

#### 陷阱：别做成 Jira

世界上不缺任务管理工具。做一个任务列表没人会从 Linear / GitHub Issues / 飞书项目迁过来，
而且会把 smelt 从「驾驶舱」拖成「项目管理工具」，定位就糊了。

**task 中心的价值不在「存任务」，在「把任务变成一个跑起来的 agent」这个动作。**

smelt 不做任务的**存储层**（那是 Linear / GitHub 的活），做任务的**执行层**：

> 一个 GitHub Issue → 点一下「我来」→ 自动建 worktree + 开会话 + 把 issue 内容喂给 agent
> → 跑完出 PR

任务从哪来无所谓（issue / Linear / 一个 markdown 队列），smelt 负责的是**从任务到 agent
的那一跳**，以及跑起来之后的全部可见性。

**与前面架构的咬合：每个任务一个 worktree** —— 撞车预警从「事后拦」升级为「结构上不可
能撞」。（worktree 集成本就在 roadmap 里躺着。）

#### 统一抽象：认领池，不是任务列表

「一个待做的任务在等人认领」和「一个卡在等审批的 agent 在等人认领」——机制上完全是同
一件事。还可以继续加：等 review 的 diff、跑挂了没人管的会话、空转半小时的 agent。

它们全都是「**一个等人处理的条目**」。所以**不该做两套 UI**，应该做**一个团队收件箱 /
认领池**：所有等人的东西都掉进来，谁有空谁点「我来」。

这个抽象比「task 中心」准确得多，也更贴定位——**驾驶舱不是管任务的，是管「什么在等我」的。**
它同时吞掉了前面的「求助广播」（第二节）。

#### 「分发给谁」的三档，建议只做前两档

1. ✅ **派给我自己的 agent**（本机）—— 基础版
2. ✅ **派给同事的 agent**（他的机器）—— 这才叫联机分发，团队价值在这
3. ❌ **自动派给空闲的机器**（谁闲跑谁）—— 会把 smelt 变成分布式任务队列 / 无人值守 CI，
   那是**另一个产品**（Devin / GitHub Actions 赛道），不要顺手滑进去

### 八、宠物串门（社交存在感）

同事的宠物跑到你桌面上蹭一下：「隔壁那个 agent 等审批 20 分钟了。」

听着像玩梗，但桌面宠物是**唯一的情绪 / 存在感通道**，是整个产品里最不可复制的部分。用它
承载 presence，比通知栏点个小红点有意思得多。

## 传输层的两个设计决定（建议现在就拍死）

### 观众不参与 resize

观战端 fit 到主人的尺寸；**只读时不反向 resize，可写接管时也不**。观众是「远程键盘」，
不是「第二块屏幕」。

这样 smeltd「attach 先回报尺寸」那条不变量**完全不用动**，绕开 tmux 那种「多 client 取
最小公共尺寸」的地狱。

### 不自建中继服务器

个人项目别去做账号体系和运维。smeltd 监听本机端口，网络问题交给用户已有的网（局域网 /
Tailscale / SSH 隧道）——unix 哲学，且**代码永远不出用户自己的网**。

### 技术骨架

- smeltd 加**可选**的 HTTP/WebSocket 端点，**默认关闭**
- 复用现有 attach 协议：`snapshot_ansi` 当首帧，PTY 字节流当增量推
- 前端一个极简 xterm.js 页面就够，别自己写终端渲染
- 一次性 token 塞 URL，可设过期、可设只读/可写

---

## ⚠️ 必须正视的安全风险

**把 PTY 暴露到网络，字面意义上就是开了个远程 shell。** 这条线一旦做错就是安全事故。

底线：
- 默认关闭
- token 强随机、可过期
- **可写模式必须主人当面点头**，不能靠 URL 携带就生效
- 绑定回环或 Tailscale 接口，**不是 `0.0.0.0`**

## 落地路径

1. **状态通道**（smeltd 结构化状态 + hook 信源 + schema）
   —— 单机即可验证，做完会话总览立刻变准，哪怕联机一行不写也不亏
2. **网络暴露**（token + WebSocket，复用 attach 协议）
   —— 底座通了，观战席是第一个前端
3. **四张脸随便挑着长**，看哪个先有人喊
